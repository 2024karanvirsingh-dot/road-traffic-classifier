01 · Public data sources

Free, keyless, available worldwide

Anything region specific, like a state DOT count program, is deliberately excluded so the identical pipeline runs anywhere. One Overpass request per city fetches everything and is cached under data/<city>/raw.json, so reruns are fully offline.

OpenStreetMap via Overpass API

Road class, lanes, speed limits, one way flags, surface, geometry. Mirror fallback for reliability.

OSM transit route relations

Bus, trolleybus, tram, and share taxi routes per road. A proven demand signal from Boston's MBTA to Nairobi's matatus.

OSM land use and amenity polygons

Commercial, retail, industrial areas plus universities, hospitals, schools, and marketplaces: trip generators.

Nominatim geocoder

City name to coordinates, so any searched city resolves to a study window.

All eight cities retrieved 2026-07-17, recorded per city in run_metadata.json. OSM changes over time, so a rerun without the cache may differ slightly.

02 · System pipeline

One command, five stages

From a city name to CSV, Markdown, and an interactive map, in a single deterministic pass.

1

Geocode

City name to a bounding box via Nominatim, or pass a bbox directly.

2

Fetch once

A single Overpass request gets roads, transit, and land use. Cached to raw.json.

3

Impute, flagged

Missing lanes and speeds fill from road class defaults, never silently.

4

Score 8 features

Each segment's score is exactly the sum of eight printed contributions.

5

Aggregate

Length weighted road rows with uncertainty fields, plus CSV, MD, and map outputs.

$ python3 classify.py "Nairobi, Kenya"
$ python3 classify.py --bbox 42.360,-71.130,42.390,-71.090 --name "Cambridge, USA"

City name mode evaluates a fixed rectangular window centered on the geocoded location, about 5.5 miles tall by default (--span to change). It does not follow municipal boundaries. Outputs per city: results.csv · results.md · map.json · run_metadata.json.

03 · Input features

Eight features per segment

Four describe built capacity, four proxy demand. Every one exists in OSM for any city on Earth.

Built capacity

F1

Functional road class

The strongest single predictor: the network role the road was built and signed for. Link roads, the ramps, scored explicitly.

F2

Lane count

Capacity that exists because demand justified building it. Values below one are treated as missing.

F3

Speed limit

Normalized to mph from any tagging style: "30 mph", a bare 50 meaning km/h, "walk", multi values.

F4

One way pairing

One way arterials are usually halves of a couplet carrying full corridor demand.

Demand proxy

F5

Transit routes

Distinct bus, tram, trolleybus, and share taxi routes on the segment. Four routes beat one. Service frequency is not yet captured.

F6

Trip generators

Segment points hashed into ~500 ft grid cells; picks up the best generator in its own or a neighboring cell. No stacking, the highest value wins.

F7

Junction connectivity

An endpoint shared by 4 or more segments marks an intersection hub, a free centrality proxy.

F8

Surface

Unpaved roads suppress through traffic. Matters greatly outside North America and Europe.

04 · Scoring algorithm

The exact formula

A plain additive score per segment. Every reported score equals the sum of the eight contributions below, and all eight are printed per road in the outputs.

class_ptsprimary + trunk link / prim. link28 / 24
class_ptssecondary / sec. link / tertiary22 / 19 / 14
class_ptsunclassified / residential8 / 5

transit_pts1 route / 2 to 3 / 4 or more+3 / +5 / +7

junction_ptsendpoint shared by 4+ segments+3

How weights were chosen

Heuristic, not statistically fitted. Road class dominates because it encodes intended network role; lanes and speed add capacity; transit, generators, and junctions proxy demand; the unpaved penalty encodes suppressed through traffic. Thresholds came from manual inspection of Cambridge corridors. The contribution columns make later calibration against real counts a simple regression.

06 · Missing data

Impute from road class, and say so

OSM tagging completeness varies enormously by region, so imputation is never silent.

Class based defaults

Missing, malformed, or nonpositive lanes and unparseable speeds fill from class defaults, e.g. residential: 1 lane, 25 mph; primary: 2 lanes, 30 mph.

Every row is flagged

Each road carries lanes_imputed_share, speed_imputed_share, and a combined data_completeness field. 1 means every value was observed.

Graceful layer absence

If transit or land use layers are empty for a region, those terms contribute zero and the class based core still ranks sensibly. Unnamed roads are scored and mapped, just excluded from named tables.

Robust fetching

Overpass mirror fallback plus response caching; an area with no supported roads raises a clear error instead of writing empty output.

07 · Aggregation & uncertainty

A road is not a single number

Road rows are length weighted: a 1 km segment counts 50 times a 20 m fragment, for the score, every feature mean, and every share. The road score is exactly the sum of its eight length weighted contribution columns, so every row stays auditable by addition. Because a named road can change character along its length, each row also ships its own uncertainty.

spread

min_segment_score, max_segment_score, and score_std expose the range behind the mean.

consistency

dominant_category + category_consistency: "High with 100% of its length High" beats an average alone.

margin

classification_margin + margin_label: distance to the nearest threshold, so a 38.1 road is never as certain as a 55.

completeness

data_completeness: how much lane and speed input was observed rather than imputed.

borderline < 2 pts moderate 2 to 5 pts stable > 5 pts

11 · Runtime and scalability

Linear cost, planet scale headroom

The only spatial operations are hash lookups on grid cells and endpoints, so cost is linear in segment count. The same code and weights ran unchanged on Cambridge, London, Paris, Tokyo, Nairobi, Sao Paulo, Mumbai, and Sydney.

12 · Limitations and improvements

What it cannot do, and the fix for each

Categories are estimated relative traffic intensity, not measured vehicles per day. Qualitative sanity checks hold across cities: Memorial Drive, McGrath Highway, and Soldiers Field Road top Cambridge; Uhuru Highway and the Nairobi Expressway top Nairobi; the Western Express Highway and the Sea Link top Mumbai. Numeric validation against counts has not been performed.

LimitationLikely effectImprovement
No measured traffic at runtimeLabels estimate likely volume, not countsCalibrate weights and thresholds against local count data where it exists
No time dimensionCannot distinguish peak and off peakIngest GTFS headways, derive peak factors from land use mix
Uneven OSM completenessLower reliability in poorly tagged areasdata_completeness flags it; betweenness centrality as a tag free signal
Route count ignores frequencyA 10 minute headway counts like an hourly oneUse GTFS service frequency where published
Name aggregation blurs long roadsA road spanning quiet and busy stretches averages outMin/max and consistency columns expose it; the map is per segment
Hand selected thresholdsCategories may not transfer perfectly between regionsFit region specific thresholds from local data
